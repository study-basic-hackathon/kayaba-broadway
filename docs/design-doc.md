# 茅場ブロードウェイ Design Doc

## 1. プロジェクト概要

### 1.1 サービス名
**茅場ブロードウェイ**

### 1.2 コンセプト
同人誌やホビーグッズをなんでも販売している商店街「中野ブロードウェイ」に感銘を受けて着想したオンラインマーケット。  
開催地・茅場町の名前を取って命名。

リアルの商店街のように **仮想空間（フィールド）を自由に歩き回り**、お店に入って商品を購入したり、店員・他のファンとコミュニケーションを取れる体験を提供する。

### 1.3 ターゲットユーザー
- 同人誌・デジタルコンテンツの購入者（バイヤー）
- ※今回のハッカソンではユーザー（購入者）側の体験に集中する

---

## 2. 機能要件

### 2.1 必須機能（MVP）

| 機能 | 説明 |
|------|------|
| 認証機能 | メールアドレス・パスワードによるログイン／ログアウト |
| ユーザー登録機能 | 新規アカウント作成 |
| ユーザー情報編集機能 | アイコン画像・表示名などプロフィール情報の変更 |
| 空間移動機能 | フィールド上でキャラクターを上下左右に移動（ポケモン式2Dマップ） |
| リアルタイム他ユーザー表示 | Partykit（WebSocket）を使い、フィールド上に他のログインユーザーのキャラクターをリアルタイム表示 |
| 購買・決済機能 | 商品をポップアップで確認→購入→デジタルファイルダウンロード |

### 2.2 できれば実装する機能（Nice to have）

| 機能 | 説明 |
|------|------|
| チャット機能 | 店内でのリアルタイムテキストチャット（店員・他ユーザーと） |
| 店内通話機能 | 店内でのWebRTCによるテレビ電話（店員・他ユーザーと） |
| 本番決済サービス統合 | Stripe等の実決済サービスへの差し替え |
| 掲示板機能 | フィールド上（店外）に設置。ユーザー同士が自由に投稿できる掲示板 |
| ライブスペース | フィールド内の自由スペースでイベント開催（WebRTC通話） |

### 2.3 今回スコープ外（将来実装予定）

- 出店者（ショップオーナー）側の管理機能
  - 商品登録・編集・削除
  - 売上管理
- 複数店舗の展開
- アバターのカスタマイズ（工数次第で検討）

---

## 3. 非機能要件

- **決済**: 初期実装はモック（購入ボタン押下で購入完了扱い）。後から実決済サービス（Stripe等）へ差し替えられる設計にする
- **アバター**: 固定キャラクターを使用（カスタマイズは工数次第で対応）
- **フィールド**: 1つの仮想フィールド内に1店舗のみ存在
- **販売商品**: 主にPDF等のデジタルコンテンツを対象とする

## 4. 画面構成

### 4.1 画面一覧

| 画面名 | 説明 |
|--------|------|
| ログイン画面 | メールアドレス・パスワードでログイン |
| アカウント登録画面 | 新規ユーザー登録フォーム |
| ユーザー情報編集画面 | アイコン・表示名などを変更 |
| フィールド画面 | 仮想空間マップ。キャラクターを操作して移動。お店へ入店可能 |
| フィールド画面（掲示板） | フィールド上（店外）に設置された掲示板。ユーザー同士が自由に投稿（Nice to have） |
| 店内画面（チャット） | 店内でのリアルタイムテキストチャット（Nice to have） |
| 店内画面（通話） | 店内でのWebRTCテレビ電話（Nice to have） |
| 店内画面（商品一覧・購入） | 商品一覧→ポップアップで詳細確認・購入・ダウンロード |
| ライブスペース画面 | フィールド内の自由スペース（時間があれば実装） |

### 4.2 画面遷移

```
[ログイン画面] ─→ [フィールド画面（移動 / 掲示板）]
     ↑                   │
[アカウント登録画面]      │ 入店
                         ↓
                    [店内（チャット / 通話 / 商品購入）]

[フィールド画面] ─→ [ライブスペース画面]
[フィールド画面] ─→ [ユーザー情報編集画面]
```

## 5. システム構成

### 5.1 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Angular + PixiJS（Cloudflare Workers でホスティング） |
| バックエンド API | Hono + Drizzle ORM（Cloudflare Workers） |
| リアルタイム通信 | Partykit（WebSocket、Cloudflare Workers） |
| ビデオ通話 | WebRTC（LiveKit もしくは STUN/TURN サーバー ※要調査） |
| データベース | Cloudflare D1（SQLite） |
| ファイルストレージ | Cloudflare R2（商品ファイル・アイコン画像） |

### 5.2 インフラ構成図

```
ブラウザ (Angular + PixiJS)
    │
    ├── REST API ──→ Cloudflare Workers (Hono + Drizzle) ──→ Cloudflare D1
    │                                                    └──→ Cloudflare R2
    │
    ├── WebSocket ──→ Partykit (Cloudflare Workers)
    │
    └── WebRTC ────→ STUN/TURN or LiveKit
```

## 6. データモデル（初期案）

> ※詳細は別途DB設計で確定する

### users（ユーザー）

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | TEXT (UUID) | 主キー |
| email | TEXT | メールアドレス（ユニーク） |
| password_hash | TEXT | パスワードハッシュ |
| display_name | TEXT | 表示名 |
| icon_url | TEXT | アイコン画像URL（R2パス） |
| created_at | INTEGER | 作成日時（Unix時刻） |

### shops（店舗）

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | TEXT (UUID) | 主キー |
| name | TEXT | 店舗名 |
| description | TEXT | 店舗説明 |
| position_x | INTEGER | フィールド上のX座標 |
| position_y | INTEGER | フィールド上のY座標 |

### products（商品）

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | TEXT (UUID) | 主キー |
| shop_id | TEXT | 所属店舗ID（FK） |
| name | TEXT | 商品名 |
| description | TEXT | 商品説明 |
| price | INTEGER | 価格（円） |
| file_url | TEXT | ダウンロードファイルURL（R2パス） |
| thumbnail_url | TEXT | サムネイル画像URL |
| created_at | INTEGER | 作成日時 |

### purchases（購入履歴）

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | TEXT (UUID) | 主キー |
| user_id | TEXT | 購入者ID（FK） |
| product_id | TEXT | 商品ID（FK） |
| purchased_at | INTEGER | 購入日時 |
| payment_status | TEXT | `mock` / `completed` / `failed` |

## 7. API エンドポイント（初期案）

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/auth/register` | ユーザー登録 |
| POST | `/auth/login` | ログイン（JWTトークン発行） |
| GET | `/users/me` | 自分のプロフィール取得 |
| PUT | `/users/me` | プロフィール更新 |
| GET | `/shops` | 店舗一覧取得 |
| GET | `/shops/:id` | 店舗詳細取得 |
| GET | `/shops/:id/products` | 店舗内商品一覧 |
| GET | `/products/:id` | 商品詳細取得 |
| POST | `/products/:id/purchase` | 商品購入（モック決済） |
| GET | `/purchases` | 自分の購入履歴一覧 |
| GET | `/purchases/:id/download` | 購入済みファイルのダウンロードURL取得 |
